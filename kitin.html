Kitin

Process Automation · Security · Go

Problem

Security findings in a codebase often stall between detection and fix. Manual remediation is slow, git history gets messy, and there is no reliable audit trail showing who changed what and when. Ad hoc database access during triage also creates unnecessary exposure.

Solution

Kitin centers on a state driven git automation engine that handles the full remediation lifecycle: scan, classify, open a tracked branch, apply a fix, and advance through review states. Every transition appends to an immutable audit log so the full history is reconstructable without rewriting past records.

Database access runs through a constrained, parameterized SQL query set designed for least privilege. Analysts get the rows they need for triage without open ended query access. On top of the automation layer, a multi model classifier trained on real exploit patterns returns structured findings (severity, category, line number, explanation, suggested fix) instead of free form LLM text.

From the terminal, you point Kitin at a file or directory and it returns prioritized results with an estimated daily exposure cost and a breakdown by severity. Findings are tagged by type (injection, auth and sessions, data exposure, hardening) so teams can route work to the right owner.

The same scan data is available through a mobile interface. The priority list surfaces high severity issues first, with issue count, daily exposure estimate, and per finding detail on tap.

Outcome

Security review becomes a tracked process with guardrails: automated git workflow, auditable state transitions, and scoped data access. Teams see what is wrong, how risky it is, and how to fix it without unstructured model output.
