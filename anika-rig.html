Custom Dynamic Rigging

Blender · Shape Key Correction · IK/FK · Character Pipeline

I built custom dynamic rigging for Anika, a free production-style character used to test the full asset pipeline from design through deformation-ready delivery. The work centers on shape key correction, IK/FK limb switching, and a driver-based skirt system with pseudo collision detection.

Modeling

I designed Anika for a short film and started with a turnaround sketch. During poly modeling I kept edge flow in mind for deformation, paying attention to pole placement and quad density so the mesh would hold up under the rig.

Rigging

I built the full armature with shape keys for both corrective deformation and facial expressions on the body and face. Control placement was chosen to be intuitive for an animator: FK for fine posing, IK where reach and planting matter, and dedicated facial sliders grouped by region.

Corrective shape keys fire on joint limits so elbows, knees, and the torso hold volume instead of collapsing. Expression shape keys cover the major phoneme and emotion shapes an animator would need on a close-up shot.

Skirt rig

I built a custom skirt system instead of relying on cloth simulation. Detection bones sit near the legs and measure rotation past a threshold. When a leg crosses that threshold, drivers push the skirt deformation bones to fold the mesh away from the limb. Separate skirt bones between the detection points smooth the curve so the fold reads as fabric rather than a single hard pivot.

Skirt rig V2 (planned)

The current system uses a single detection bone per side, which produces a global skirt response. Asymmetric leg motion, like lifting one knee, moves the whole skirt instead of just the affected panel. The planned fix is radial segmentation into 8 to 12 vertical bone chains with angular weighted drivers so each panel responds independently to the nearest leg.

Shaders and texturing

After rigging I moved to look development. I added seams, laid out UVs, and painted normals in Krita for a hand-painted surface read. I then hooked up the painted normal map to the character BSDF.

For the hair I built a custom shader driven by the y axis of the UV. A Perlin noise node creates differentiating lines on the UV, and a color ramp generates a band around the middle of the hair. That band follows the camera direction so the highlight shifts higher when the camera is higher and lower when it is lower. A color mix node blends hair colors and passes the result into a BSDF so the highlight reacts to scene lighting.

Hair shader progress

For Anika's dress I wanted a painted silk look. I used the mesh normal as a vector input to a Voronoi texture, which produces broad color bands on flat areas and tighter bands on complex geometry. I then used reflection with a color ramp so the more reflective regions read metallic, matching how silk with retained sericin catches light. A final color mix adds a blue tint to the darker regions.

Downstream considerations

The rig hierarchy separates deformation bones from control bones so an animator only touches the control layer. IK/FK switches live on the limb roots, and facial controls are grouped under a single face master for show/hide in the outliner. Shape keys are named by region and function (corrective_elbow_L, expr_smile, etc.) so a pipeline script or importer can filter them by prefix.

An animator needs the control rig visible, IK/FK set per limb before animation, and the skirt detection thresholds tuned if the character wears a long skirt in a sitting or walking cycle. For export, apply the armature modifier and bake shape keys if the target engine does not support them live.
