Bouncing Ball

OpenGL · GLSL · C++ · GPU Physics

Problem

Simulating hundreds of colliding bodies on the CPU means serial loops over every pair each frame. That does not scale, and it keeps physics off the same hardware that renders the scene.

Solution

An OpenGL simulation that parallelizes ball physics on the GPU. Each ball's position and velocity live in a 1×N floating point texture, updated every frame by a physics fragment shader. Collisions use conservation of momentum; wall bounces keep balls inside the viewport.

Ball state stored in a texture

Every ball is packed into one row of an RGBA32F texture: R and G hold position, B and A hold velocity. C++ initializes random positions and velocities, then ping-pongs between two textures each frame so the physics shader always reads last frame's state and writes the next. 

// each ball = one texel: (x, y, vx, vy)
ballData[i*4 + 0] = ...; // x
ballData[i*4 + 1] = ...; // y
ballData[i*4 + 2] = cos(angle) * speed; // vx
ballData[i*4 + 3] = sin(angle) * speed; // vy

GLuint tex[2];
tex[0] = makeStateTexture(NUM_BALLS, ballData.data());
tex[1] = makeStateTexture(NUM_BALLS, nullptr);

Physics pass, one fragment per ball

A fullscreen quad is drawn to an offscreen framebuffer sized NUM_BALLS × 1. Each fragment looks up its own ball with gl_FragCoord.x, reads state from the input texture, resolves collisions against every other ball, bounces off walls, applies damping, and integrates position forward by delta time. 

int i = int(gl_FragCoord.x);
vec4 self = texelFetch(uState, ivec2(i, 0), 0);
vec2 pos = self.xy;
vec2 vel = self.zw;

Ball-to-ball collision and conservation of momentum

For each pair of overlapping balls, the shader computes the collision normal and checks whether they are moving toward each other. If so, velocity is projected along the normal and reflected. That impulse conserves momentum for equal-mass spheres colliding elastically along the contact normal. 

vec2 diff = pos - other.xy;
float dist = length(diff);
if (dist < minDist && dist > 0.0001) {
  vec2 n = diff / dist;
  float dvDot = dot(vel - v2, n);
  if (dvDot < 0.0) {
    vel -= dvDot * n;
  }
}

Because each ball's fragment runs independently and writes only its own output, ball j applies the same logic when it processes ball i. Both velocities update in parallel without a serial loop on the CPU.

Wall bounce and integration

After ball collisions, the shader clamps position inside the bounds and flips velocity to bounce off each wall. A small damping factor prevents energy from growing unbounded, then position is integrated with pos += vel * uDelta. 

if (pos.x - uRadius < -1.0) { pos.x = -1.0 + uRadius; vel.x = abs(vel.x); }
if (pos.x + uRadius >  1.0) { pos.x =  1.0 - uRadius; vel.x = -abs(vel.x); }
// ... same for y walls ...

vel *= 0.99999;
pos += vel * uDelta;
newState = vec4(pos, vel);

C++ drives the ping-pong each frame. It binds the write FBO, runs the physics program, swaps textures, then renders. 

glBindFramebuffer(GL_FRAMEBUFFER, fbo[1 - current]);
glViewport(0, 0, NUM_BALLS, 1);
glUseProgram(physProg);
glBindTexture(GL_TEXTURE_2D, tex[current]);
glDrawArrays(GL_TRIANGLES, 0, 6);
current = 1 - current;

Render pass with instanced draw for all balls

A separate render shader draws every ball in one instanced call. gl_InstanceID selects which texel to read; the unit quad is scaled by radius and offset to the ball's position. The fragment shader discards corners outside the circle and assigns each ball a unique color from a hash of its index. 

// render.vert.glsl
vec4 state = texelFetch(uState, ivec2(gl_InstanceID, 0), 0);
vec2 worldPos = state.xy + aQuadPos * uRadius;
gl_Position = vec4(worldPos, 0.0, 1.0);

// main.cpp, one draw call for every ball
glDrawArraysInstanced(GL_TRIANGLES, 0, 6, NUM_BALLS);
// render.frag.glsl
float dist = length(vLocalPos);
if (dist > 1.0) discard;

float r = fract(sin(id * 127.1) * 43758.5);
float g = fract(sin(id * 269.5) * 23421.6);
float b = fract(sin(id * 419.2) * 67890.3);
FragColor = vec4(r, g, b, 1.0);

Outcome

Hundreds of balls collide and render in real time with one physics pass and one instanced draw call. State stays on the GPU between frames through texture ping ponging, with no per ball CPU loop.
