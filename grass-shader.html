Grass Shader

Unity · Compute Shader · HLSL · C#

A Unity URP grass system that generates all blade geometry on the GPU in parallel using a compute shader — one thread per source triangle, output streamed to an append buffer, drawn in a single indirect draw call.

One blade per triangle

Each thread in the compute kernel handles one triangle from the source mesh. It finds the triangle center in world space, builds a tangent-to-world matrix from the triangle edges, and sprouts a blade from that point. Blade width and height are randomized per position using randNegative1to1.

Building sections from a single strip of triangles

A blade is not one triangle, it is a strip of square sections stacked along the height of the blade. For each segment, the compute shader places two vertices at the same height v, then adds a final tip vertex. Adjacent groups of three vertices become the draw triangles appended to the buffer.

As v rises toward the tip, the horizontal offset u converges on 0.5, tapering the blade. Segment count is reduced at distance via LOD so far blades use fewer sections. 

for (int i = 0; i < numBladeSegments; i++) {
  float v = i / (float)numBladeSegments;
  float u = 0.5 - (1 - v) * 0.5;
  float3x3 transform = FigureTransformationForHeight(v, twistMatrix, tsToWs, maxBend, windAxis, windStrength);

  vertices[i * 2]     = SetupBladePoint(positionWS, dimensions, transform, float2(u, v), windStrength);
  vertices[i * 2 + 1] = SetupBladePoint(positionWS, dimensions, transform, float2(1 - u, v), windStrength);
}

float3x3 tipTransform = FigureTransformationForHeight(1, twistMatrix, tsToWs, maxBend, windAxis, windStrength);
vertices[numBladeSegments * 2] = SetupBladePoint(positionWS, dimensions, tipTransform, float2(0.5, 1), windStrength);

int numTriangles = (numBladeSegments - 1) * 2 + 1;
for (i = 0; i < numTriangles; i++) {
  DrawTriangle tri = (DrawTriangle)0;
  tri.vertices[0] = vertices[i];
  tri.vertices[1] = vertices[i + 1];
  tri.vertices[2] = vertices[i + 2];
  _DrawTriangles.Append(tri);
}

Each vertex offset starts in tangent space as a width/height pair, then is rotated into world space and added to the anchor point on the terrain triangle. 

float3 offsetTS = float3((uv.x - 0.5f) * dimensions.x, 0, uv.y * dimensions.y);
float3 offsetWS = mul(tsToWs, offsetTS);
float3 positionWS = anchorWS + offsetWS;

Rotating the blade — bend, twist, and wind

Rotation is applied in order: bend, twist, convert to world space, then wind. Bend increases toward the tip using pow(v, _BladeCurvature). A random twist around the blade axis gives each blade a unique facing direction. Wind rotation uses the same height multiplier v, so the base stays anchored and the tip moves the most. 

float3x3 bendMatrix = AngleAxis3x3(maxBend * pow(v, _BladeCurvature), float3(1, 0, 0));
float3x3 windMatrix = AngleAxis3x3(_WindAmplitude * v * windStrength, windAxis);
return mul(windMatrix, mul(tsToWs, mul(twistMatrix, bendMatrix)));

Procedural wind noise

Wind strength is sampled per blade in world space. The compute shader uses a cheap gradient noise function — hash values at grid corners blended with a quintic curve for smooth interpolation — then scrolls the sample position over time along a wind direction. 

float gradientNoise(float2 p) {
  float2 i = floor(p);
  float2 f = frac(p);
  float2 u = f * f * f * (f * (f * 6.0 - 15.0) + 10.0);
  // ... hash corners, lerp ...
}

float GetWindStrength(float3 positionWS) {
  float2 windDir = normalize(float2(1, 0.3));
  float2 windUV = (positionWS.xz * _WindPosMult) * 0.2;
  windUV -= windDir * _Time.y * _WindTimeMult;
  return gradientNoise(windUV);
}

That noise value becomes windStrength on each vertex. It is multiplied by _WindAmplitude and height v inside FigureTransformationForHeight, controlling how far the blade rotates around the wind axis. The wind axis itself is perpendicular to the global wind direction. 

float3 GetWindAxis(float3 positionWS, float3 normalWS) {
  float3 windDir3D = float3(1, 0, 0.3);
  return normalize(cross(float3(0, 1, 0), windDir3D));
}

Fragment shading — light, shadow tint, and wind darkening

The graphics shader reads each generated triangle from a structured buffer. The fragment shader lerps from base color to tip color by height, then darkens albedo where wind strength is high. Directional light and shadow maps set how lit each pixel is; the base of each blade is darkened further. Shadowed areas blend toward a separate shadow tint color. 

float3 albedo = lerp(_BaseColor.rgb, _TipColor.rgb, input.uv);
albedo *= lerp(1.0, 0.5, input.windStrength * 0.5);

float NdotL = saturate(dot(normalWS, mainLight.direction));
float lightAmount = NdotL * mainLight.shadowAttenuation;
lightAmount *= input.uv;

float3 litColor = albedo * mainLight.color.rgb;
float3 shadowedColor = lerp(albedo, _ShadowColor.rgb, _ShadowTintStrength);
float3 finalColor = lerp(shadowedColor, litColor, lightAmount);

The wind darkening on albedo simulates self-shadowing as blades lean — a bright tip tilting away reads slightly darker before the main light calculation even runs.

C# renderer — feeding the compute shader and drawing

ProceduralGrassRenderer uploads the source mesh into compute buffers, passes all grass and wind settings each frame, dispatches one thread group per source triangle, then draws the append buffer with Graphics.DrawProceduralIndirect. 

instantiatedGrassComputeShader.SetFloat("_WindAmplitude", grassSettings.windAmplitude);
instantiatedGrassComputeShader.SetFloat("_WindTimeMult", grassSettings.windPeriod);
instantiatedGrassComputeShader.SetFloat("_WindPosMult", grassSettings.windScale);
instantiatedGrassComputeShader.SetTexture(idGrassKernel, "_WindNoiseTexture", grassSettings.windNoiseTexture);

drawBuffer.SetCounterValue(0);
instantiatedGrassComputeShader.Dispatch(idGrassKernel, dispatchSize, 1, 1);
Graphics.DrawProceduralIndirect(instantiatedMaterial, bounds, MeshTopology.Triangles, argsBuffer, 0,
    null, null, ShadowCastingMode.Off, true, gameObject.layer);

Custom Unity editor — grass settings and wind texture generator

ProceduralGrassRendererEditor extends the default inspector for the renderer. All grass settings — segment count, blade width/height, curvature, bend, wind amplitude, and LOD distances — appear as standard Unity fields through DrawDefaultInspector(). Below that, a custom panel generates a tileable Perlin noise texture for wind. 

[CustomEditor(typeof(ProceduralGrassRenderer))]
public class ProceduralGrassRendererEditor : Editor {
  public override void OnInspectorGUI() {
    DrawDefaultInspector();
    noiseScale = EditorGUILayout.Slider("Noise Scale", noiseScale, 1f, 32f);
    noiseSeed = EditorGUILayout.IntField("Seed", noiseSeed);
    if (GUILayout.Button("Generate Wind Noise Texture")) {
      GenerateAndAssignNoiseTexture();
    }
  }
}

The generator fills a 512×512 texture with two independent tileable Perlin noise channels (R and G) using Mathf.PerlinNoise, saves it as WindNoise.png, and assigns it to grassSettings.windNoiseTexture on the renderer. 

float r = TileablePerlin(u, v, noiseScale, offsetX, offsetY);
float g = TileablePerlin(u, v, noiseScale, offsetX2, offsetY2);
pixels[y * TEXTURE_SIZE + x] = new Color(r, g, 0f, 1f);

SerializedProperty prop = so.FindProperty("grassSettings.windNoiseTexture");
prop.objectReferenceValue = loadedTexture;
