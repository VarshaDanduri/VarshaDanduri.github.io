Multimodal Clinical Data Pipeline

Data Pipeline · Python · Multimodal ML

Problem

The MIMIC-IV Data Pipeline is an open source project (300+ GitHub stars) that helps clinical researchers train models on hospital data. The original release handled structured EHR data only. Imaging, ECG waveforms, and clinical notes lived in separate stores with incompatible schemas, so multimodal research required researchers to rebuild ingestion and alignment themselves.

Solution

I extended the pipeline to ingest five modalities through a shared preprocessing framework: structured vitals and labs, clinical notes, waveforms, chest X-rays, and echocardiograms. Each modality gets its own cleaning and normalization step, but outputs conform to the existing downstream interface so notebooks and training modules that already consumed structured data did not break.

Multi format ingestion handles modality specific schemas: ICD based cohort filtering and imputation for structured data, temporal binning for sequential records, and alignment logic that connects text, waveform, and imaging data to the same patient timeline. Automatic training support runs after preprocessing, with configurable architectures and evaluation including optional fairness reporting.

Data cleaning and cohort building

Users select modalities, filter by ICD codes or disease name, remove outliers, impute missing values, and group clinical features using standard coding systems. Each step is configurable and recorded for reproducibility.

Temporal alignment

Sequential clinical data is binned into equal length time intervals and filtered by time series length according to user preference, producing aligned datasets that sequential models can consume directly.

Outcome

Researchers can pick a cohort, preprocess across modalities, train, and export from a single notebook workflow (mainPipeline.ipynb). I am a co author on the resulting paper. The extension ships as part of the upstream repo used by the broader clinical ML community.
